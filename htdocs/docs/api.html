Empire API Documentation

The Empire API attempts to use the REST style of doing things. Methods that don't change the state of the system use GET. Methods that do use POST. URL for API: http://cartiki.com/api/

Methods

createLocation

This function only accepts its parameters via POST because it is changing state on the server. Only the name parameter is required.
Parameters: name, connector, parentLocId

getLocationsByIds

Returns an XML document describing as many locIds as you can fit in the URL. LocIds should be separated with a comma.
Parameters: locIds

getLocationsByName

Returns an XML document describing locations whose name contains the name parameter string.
Parameters: name

getLocationsByGps

Returns an XML document describing locations ordered by area which contain the given x/y (long/lat) coordinate.
Parameters: x, y

getLocationBoundsOnMap

Returns an XML document with name and geographical bounds of the given locIds. LocIds should be separated with a comma.
Parameters: locIds
